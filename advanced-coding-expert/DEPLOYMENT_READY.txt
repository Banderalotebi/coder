🔧 CUSTOMIZATION EXAMPLES:
═══════════════════════════════════════════════════════════════════════════════

Example 1: Use Larger Model
──────────────────────────
ollama pull llama2:13b
# Edit config.yaml: model: "llama2:13b"
# 13B = better answers but slower


Example 2: Fine-tune on Your Code Style
────────────────────────────────────────
# Collect successful code reviews (winners + losers)
# Save as data/training_data.jsonl
finetuner = UnslothCodeFineTuner()
finetuner.load_model()
finetuner.setup_lora()
dataset = finetuner.load_training_data("custom")
finetuner.train(dataset)
# Now Llama knows your style!


Example 3: Add Custom Tools
────────────────────────────
class MyCustomTool:
    def analyze_security(self, file_path):
        # Your security analysis logic
        pass

agent.tools["security"] = MyCustomTool()
# Agent can now use your tools!


Example 4: Deploy to Production
───────────────────────────────
# Convert to GGUF for Ollama:
python convert.py ./models/llama-expert --outfile model.gguf
ollama create coding-expert -f Modelfile
ollama run coding-expert

# Or deploy via Docker:
FROM ollama/ollama
COPY ./models/llama-expert /models/
EXPOSE 11434

⚡ QUICK COMMANDS REFERENCE:

ollama serve &

# Query via API

  -H "Content-Type: application/json" \
  -d '{"question":"How does this work?"}'

# Complete a task with tools

  -H "Content-Type: application/json" \
  -d '{"task":"Find bugs", "use_tools":true}'

# Index your code
curl -X POST http://localhost:8000/index \
  -H "Content-Type: application/json" \
  -d '{"directory":"/path/to/project"}'

# Direct search
curl http://localhost:8000/search?q=database&k=5

# Check health
curl http://localhost:8000/health
